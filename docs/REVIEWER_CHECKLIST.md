# Reviewer Checklist

Use this checklist for one short external review pass. Reviewers should answer
only the sections relevant to their background.

## Links To Open

- README
- live app
- model card
- validation report
- API guide if you are reviewing developer usability

## First-Impression Check

- In under 60 seconds, can you tell what the project does?
- Is it obvious that this is a research and portfolio demo, not a clinical tool?
- Does the project feel polished enough to share with an employer, collaborator, or technical reviewer?

## App Review

- Is the calculator easy to use on first open?
- Is the risk score explanation clear?
- Is the PHQ-9-equivalent wording understandable and not misleading?
- Are the caveats and safe-use notes visible enough?
- Is there any confusing label, chart, or interaction?

## Documentation Review

- Does the README explain the project quickly?
- Can you find the live demo, architecture, and key limitations without hunting?
- Are the model card and validation report honest and readable?
- Is anything overclaimed, underexplained, or hard to trust?

## Technical Review

- Could you tell how to run the project locally?
- Is the repo structure understandable?
- Do the API docs feel usable?
- Are calibration, subgroup evaluation, and error analysis documented clearly enough?
- What is the biggest technical credibility gap still visible?

## Safe-Use And Claims Review

- Are the non-clinical disclaimers strong enough?
- Does anything read as too close to a diagnostic or clinical claim?
- Are the risk labels and threshold explanations responsible?

## Final Questions

- What is the strongest part of the project?
- What is the weakest part of the project?
- What one change would most improve it in the next week?
- Would you feel comfortable sharing this project with another technical reviewer? Why or why not?
